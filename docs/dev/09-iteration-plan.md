# 迭代开发计划

## 迭代原则

- 先打通账号、社区、频道、消息和权限主链路，再补齐通知、语音状态和审计。
- 每个迭代都必须形成可运行、可联调、可验收的成果。
- P0 范围优先，P1 只在 P0 稳定后进入。
- 文档、接口契约、数据模型和测试用例随功能同步更新。

## 软件工程方法

本项目采用以敏捷迭代为主，原型法、需求追踪、模块化开发和验收驱动测试相结合的软件工程方法。开发过程围绕 P0、P1 和 P2 优先级组织，优先完成具备演示和交付价值的 P0 MVP 范围，再根据时间与质量情况迭代补充 P1 增强能力。

- 敏捷迭代开发：按照账号、社交与私聊、社区与频道、消息与附件、通知与未读、权限和语音状态等模块划分迭代任务。每轮迭代应形成可运行、可联调、可验收的阶段成果。
- 原型法：在实现核心界面和流程前，先通过页面原型确认社区列表、频道导航、消息收发、权限配置和通知提醒等关键交互，降低需求理解偏差和返工风险。
- 需求追踪：以功能需求编号为主线，维护需求、开发模块、接口与事件、关键实体和验收编号之间的映射关系，保证 P0 需求均能追踪到实现和测试依据。
- 模块化开发：各模块保持清晰职责边界，公共权限校验、实时事件分发、日志审计和配置限制作为跨模块支撑能力统一处理，避免重复实现和规则不一致。
- 验收驱动测试：开发完成后以验收编号、异常流程、权限矩阵和非功能指标作为测试依据，重点检查注册登录、社区加入、频道消息、角色权限、通知未读和语音状态同步等核心场景。
- 版本控制与联调：使用 Git 管理代码版本，按功能或模块进行阶段性提交与合并，并在每轮迭代后开展接口联调和回归检查，降低集成风险。

## M1 工程与账号基础

交付：

- 初始化 pnpm workspace、React/Vite、NestJS、Prisma、Docker Compose。
- 建立 PostgreSQL、Redis、MinIO 本地服务。
- 实现用户注册、登录、刷新会话、退出登录。
- 实现个人资料读取和修改。
- 建立统一错误格式、请求 ID、基础审计。

验收：

- 新用户能注册、登录并获取用户摘要。
- 错误密码不能创建会话。
- 密码不会明文出现在数据库或日志中。

## M2 社交与社区骨架

交付：

- 实现好友申请、接受、拒绝和好友列表。
- 好友接受后创建或复用一对一私聊会话。
- 实现社区创建，自动生成所有者、默认角色和默认文本频道。
- 实现邀请加入和主动退出社区。

验收：

- 用户能添加好友并打开私聊。
- 用户能创建社区并通过邀请码加入。
- 所有者未转移所有权前不能退出社区。

## M3 频道、消息和未读

交付：

- 实现频道创建、编辑、排序和删除。
- 实现文本频道和私聊消息发送。
- 实现消息历史游标分页。
- 实现 `ReadState`、未读计数和通知基础记录。
- 实现附件初始化、完成和受保护访问。

验收：

- 有权成员能在文本频道发送消息，其他有权成员可加载历史。
- 空消息、超长消息、非法附件被拒绝。
- 用户进入频道后未读可更新。

## M4 权限与管理

交付：

- 实现角色创建、修改、删除和成员角色分配。
- 实现频道权限覆盖和统一 `CheckPermission`。
- 将权限守卫接入频道、消息、附件、成员和角色接口。
- 实现成员移除、禁言、恢复。
- 实现消息撤回和管理员删除。

验收：

- 无权成员不能查看受限频道或发送消息。
- 普通成员不能删除他人消息、管理成员或分配角色。
- 管理员不能管理高于自身权限的成员。

## M5 实时与语音状态

交付：

- 实现 `/realtime` Socket.IO 命名空间。
- 实现用户房间、私聊房间、社区房间、频道房间和语音房间。
- 发布 `MessageCreated`、`UnreadUpdated`、`NotificationCreated`、`PresenceChanged`、`PermissionChanged`。
- 实现语音频道加入、退出、静音、闭麦和断线释放。
- 实现重连补偿流程。

验收：

- 文本消息正常网络下 1 秒内可见。
- 未读和通知 2 秒内同步。
- 语音状态 3 秒内同步。
- 断线超时后在线状态和语音会话释放。

## M6 验收加固

交付：

- 编写端到端测试覆盖核心验收路径。
- 编写权限矩阵测试和异常流程测试。
- 加入基础压测脚本和日志追踪检查。
- 完善演示数据和本地启动说明。

验收：

- AC-01 至 AC-06 全部通过。
- AC-E1 至 AC-E8 全部通过。
- AC-N1 至 AC-N5 有可执行验证方式。

## 联调顺序

1. 鉴权和统一错误格式。
2. 用户资料和会话摘要。
3. 社区、频道和成员查询。
4. 消息发送和历史加载。
5. 未读与通知。
6. 权限守卫和频道覆盖。
7. Socket 连接和事件分发。
8. 语音状态。
9. 附件访问控制。
10. 端到端验收。

## P1 延后能力

| 能力 | 进入条件 |
|---|---|
| 密码找回与重置 FR-03 | M1 稳定且验证通道可用。 |
| 离线推送 | 站内通知稳定后接入。 |
| 社区搜索增强 | 社区创建、加入和成员管理稳定后接入。 |
| 审计查询界面 | 审计写入稳定后接入。 |
| 更多消息管理 | 基础删除、撤回和权限测试稳定后接入。 |

## 不进入 v1 的能力

- 真实多人语音媒体流。
- 屏幕共享和直播。
- 机器人平台和开放接口市场。
- 商业化订阅、广告投放和付费会员。
- 复杂内容审核和独立运营后台。
