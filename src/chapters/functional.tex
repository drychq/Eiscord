\section{功能需求}

本章采用开发可追踪的需求格式描述系统功能。每条需求均给出优先级、参与者、触发条件、前置条件、主流程、异常流程和开发追踪信息。开发追踪中的接口、事件、实体和验收编号分别对应第 4 章和第 6 章内容。

\subsection{账号与身份认证}
本模块负责用户生命周期的起始环节，为后续好友关系、社区管理和实时交互提供统一的身份基础。

\Requirement{FR-01}{用户注册}{P0}{
访客用户
}{
用户提交注册表单。
}{
用户未登录；邮箱或手机号、用户名未被平台占用；密码满足安全规则。
}{
系统校验输入格式与唯一性；创建用户账号和基础资料；记录注册时间、账号状态和默认在线状态；返回注册结果，并在需要时生成验证任务。
}{
邮箱、手机号或用户名重复时拒绝注册；密码不符合规则时返回校验错误；验证服务不可用时允许账号进入待验证状态并提示用户稍后完成验证。
}{
接口：\texttt{RegisterUser}；事件：\texttt{NotificationCreated}；实体：User、AuditLog；验收：AC-01、AC-E1。
}

\Requirement{FR-02}{登录与身份验证}{P0}{
注册用户
}{
用户提交登录凭证。
}{
账号已存在且处于可登录状态；用户提供用户名、邮箱或手机号与密码等合法凭证。
}{
系统校验账号状态和密码；创建有效会话；返回用户资料摘要、好友状态、社区列表、未读数和通知摘要；记录登录审计信息。
}{
账号不存在、密码错误、账号禁用或验证未完成时拒绝登录并返回明确原因；连续失败应记录安全审计信息，不得泄露明文密码或完整敏感凭证。
}{
接口：\texttt{LoginUser}；事件：\texttt{PresenceChanged}、\texttt{UnreadUpdated}；实体：User、ReadState、Notification、AuditLog；验收：AC-01、AC-E2。
}

\Requirement{FR-03}{密码找回与重置}{P1}{
注册用户
}{
用户无法使用原密码登录并发起找回流程。
}{
用户提供与账号绑定的邮箱、手机号或等效验证信息；验证通道可用。
}{
系统生成一次性验证凭据；用户验证通过后设置新密码；旧密码和旧重置凭据失效；系统记录重置审计信息。
}{
验证凭据错误、过期、重复使用或账号不存在时拒绝重置；重置成功后不得继续接受旧密码登录。
}{
接口：\texttt{ResetPassword}；事件：\texttt{NotificationCreated}；实体：User、AuditLog；验收：AC-01。
}

\subsection{个人资料、在线状态与社交关系}
本模块描述用户在平台层面的个人信息维护、状态展示和一对一社交行为。

\Requirement{FR-04}{个人资料维护}{P0}{
注册用户
}{
用户查看或提交个人资料变更。
}{
用户已成功登录；待修改字段满足长度、格式和文件限制。
}{
系统展示用户昵称、头像、个性签名等资料；保存合法修改；在好友列表、私聊窗口、社区成员列表等展示位置同步资料摘要。
}{
头像文件不符合限制、昵称为空或字段超长时拒绝保存；无效资料不得污染已展示的用户资料。
}{
接口：\texttt{UpdateProfile}；事件：\texttt{PresenceChanged}；实体：User、Attachment、AuditLog；验收：AC-01。
}

\Requirement{FR-05}{在线状态同步}{P0}{
注册用户、社区成员、好友
}{
用户登录、断开连接、切换状态或连接超时。
}{
用户拥有有效会话；客户端与服务端存在可识别连接或最近心跳记录。
}{
系统维护在线、离开、忙碌、隐身和离线状态；将可见状态同步到好友列表、私聊窗口和同社区成员列表；隐身状态对他人表现为离线。
}{
连接异常断开时应在超时后转为离线或离开；用户无权查看目标资料时不得接收其详细状态。
}{
接口：\texttt{UpdatePresence}；事件：\texttt{PresenceChanged}；实体：User、Membership；验收：AC-06、AC-E8。
}

\Requirement{FR-06}{好友申请与处理}{P0}{
注册用户
}{
用户搜索目标用户并发送、接受、拒绝或忽略好友申请。
}{
双方均为平台注册用户；双方尚未建立有效好友关系；操作者有权处理对应申请。
}{
系统创建好友申请并通知接收方；接收方接受后建立好友关系；拒绝或忽略后更新申请状态；双方好友列表及时反映最新关系。
}{
重复申请、向自己申请、目标不存在或处理他人申请时拒绝操作；已删除关系再次申请时应创建新的待处理关系或复用可恢复状态。
}{
接口：\texttt{CreateFriendRequest}、\texttt{AcceptFriendRequest}；事件：\texttt{NotificationCreated}、\texttt{PresenceChanged}；实体：Friendship、Notification、AuditLog；验收：AC-01、AC-E3。
}

\Requirement{FR-07}{私聊会话}{P0}{
好友用户
}{
好友用户打开私聊入口或发送私聊消息。
}{
双方已建立好友关系；发送方已登录；会话未被权限规则阻止。
}{
系统创建或复用一对一私聊会话；加载最近消息和未读位置；允许双方发送文本消息、附件元数据和提及；更新最近会话时间与未读计数。
}{
非好友发起私聊、会话参与者不匹配或发送内容不合法时拒绝操作；被删除或撤回的消息不得以普通消息展示。
}{
接口：\texttt{SendMessage}；事件：\texttt{MessageCreated}、\texttt{UnreadUpdated}、\texttt{NotificationCreated}；实体：DirectConversation、Message、Attachment、ReadState、Notification；验收：AC-01、AC-05。
}

\subsection{社区与成员管理}
本模块定义围绕社区空间的创建、邀请、加入、退出和成员基础管理需求。

\Requirement{FR-08}{创建社区}{P0}{
注册用户
}{
用户提交社区创建表单。
}{
用户已登录；社区名称、图标和简介满足输入限制；用户未超出可创建社区数量限制。
}{
系统创建社区；将创建者设为社区所有者；创建默认成员角色、默认文本频道和必要的权限配置；返回社区详情并进入社区视图。
}{
社区名称非法、图标不符合限制或创建数量超限时拒绝创建；默认角色或默认频道创建失败时不得留下不完整社区。
}{
接口：\texttt{CreateServer}；事件：\texttt{MemberJoined}、\texttt{ChannelChanged}、\texttt{PermissionChanged}；实体：Server、Membership、Role、Channel、AuditLog；验收：AC-02。
}

\Requirement{FR-09}{加入、邀请与退出社区}{P0}{
注册用户、社区成员、社区管理员
}{
用户使用邀请码或邀请链接加入社区，或成员主动退出社区。
}{
用户已登录；邀请有效且未过期；用户未被社区限制访问；退出操作者为本人或具备成员管理权限。
}{
系统校验邀请和访问限制；加入成功后创建成员关系并分配默认角色；退出成功后移除成员访问权限并更新成员列表。
}{
无效邀请、过期邀请、重复加入、被封禁用户加入或社区所有者直接退出时拒绝操作；退出后不得继续访问受限频道和历史成员权限。
}{
接口：\texttt{JoinServer}、\texttt{LeaveServer}；事件：\texttt{MemberJoined}、\texttt{PermissionChanged}、\texttt{UnreadUpdated}；实体：Invitation、Membership、Role、AuditLog；验收：AC-02、AC-E4。
}

\Requirement{FR-10}{社区成员列表与基础管理}{P0}{
社区成员、社区管理员、社区所有者
}{
用户进入社区成员列表，或管理员执行成员移除、禁言、恢复等基础管理动作。
}{
用户已加入目标社区；管理动作的操作者具备相应权限；目标成员存在。
}{
系统按权限展示成员列表、成员资料摘要、在线状态和角色信息；管理者可对成员执行允许的基础管理动作；系统同步成员和权限变化。
}{
普通成员不得执行管理动作；管理员不得移除社区所有者或高于自身权限的成员；目标成员不存在时返回失败并记录审计。
}{
接口：\texttt{ManageMember}；事件：\texttt{MemberChanged}、\texttt{PermissionChanged}、\texttt{PresenceChanged}；实体：Membership、Role、AuditLog；验收：AC-04。
}

\subsection{频道管理}
本模块定义社区内部文本频道和语音频道的组织方式及访问控制要求。

\Requirement{FR-11}{频道创建与配置}{P0}{
社区管理员、社区所有者
}{
具备权限的用户创建、编辑、排序或删除频道。
}{
用户已加入目标社区；拥有频道管理权限；频道名称、类型和排序配置合法。
}{
系统创建文本频道或语音频道；保存名称、主题、排序、可见性和权限覆盖规则；向相关成员同步频道列表变化。
}{
频道类型非法、名称重复或操作者权限不足时拒绝操作；删除频道后应阻止继续发送消息或加入语音状态。
}{
接口：\texttt{CreateChannel}；事件：\texttt{ChannelChanged}、\texttt{PermissionChanged}；实体：Channel、PermissionOverwrite、AuditLog；验收：AC-02、AC-04。
}

\Requirement{FR-12}{频道访问控制}{P0}{
社区成员、社区管理员、社区所有者
}{
用户查看频道、发送消息、加入语音频道或管理员调整频道权限。
}{
目标社区存在成员、角色和频道权限规则；用户发起需要权限判断的操作。
}{
系统根据成员角色、频道覆盖规则和所有者特权计算有效权限；仅向有权用户展示频道和内容；权限变更后立即影响后续操作。
}{
权限不足时拒绝访问、发言、删消息或加入语音状态；不得通过历史链接、附件地址或实时事件绕过权限判断。
}{
接口：\texttt{CheckPermission}；事件：\texttt{PermissionChanged}；实体：Role、PermissionOverwrite、Membership、Channel、AuditLog；验收：AC-04、AC-E5、AC-E7。
}

\subsection{文本消息}
本模块聚焦文本频道和私聊中的消息生产、分发、历史查看、未读定位及内容管理要求。

\Requirement{FR-13}{文本消息发送与附件上传}{P0}{
好友用户、社区成员
}{
用户在私聊或文本频道中发送消息。
}{
用户已登录；拥有当前会话发送权限；文本内容、附件类型和附件大小满足平台限制。
}{
系统保存消息内容、附件元数据和提及列表；将消息实时分发给会话参与者或有权频道成员；为被提及用户和离开当前会话的用户生成通知与未读数。
}{
空消息、超长消息、非法附件、无权发送或目标会话不存在时拒绝发送；失败消息不得被广播或计入未读。
}{
接口：\texttt{SendMessage}；事件：\texttt{MessageCreated}、\texttt{UnreadUpdated}、\texttt{NotificationCreated}；实体：Message、Attachment、ReadState、Notification、AuditLog；验收：AC-02、AC-05、AC-E6。
}

\Requirement{FR-14}{历史消息加载与未读定位}{P0}{
好友用户、社区成员
}{
用户进入私聊或文本频道，或滚动加载历史消息。
}{
目标会话存在；用户对该会话或频道拥有查看权限；存在可查询的消息记录或空会话状态。
}{
系统按时间或游标分页返回历史消息；过滤已删除内容和无权查看内容；定位最近未读位置；用户阅读后更新读取状态和未读数。
}{
无查看权限、游标无效或会话不存在时拒绝返回消息；用户退出社区后不得继续加载该社区受限频道历史。
}{
接口：\texttt{LoadMessages}、\texttt{MarkRead}；事件：\texttt{UnreadUpdated}；实体：Message、ReadState、Membership、DirectConversation；验收：AC-02、AC-05、AC-E5。
}

\Requirement{FR-15}{消息撤回与删除}{P0}{
消息发送者、社区管理员、社区所有者
}{
用户撤回自己的消息，或管理员删除频道内违规消息。
}{
目标消息存在；操作者是消息发送者或拥有消息管理权限；消息所属会话仍可识别。
}{
系统更新消息可见状态；保留必要的删除或撤回审计信息；向相关客户端同步消息变更；更新相关通知或未读状态。
}{
操作者无权、目标消息不存在、消息已删除或消息属于无权管理的频道时拒绝操作；私聊中不得删除对方历史记录的服务端存档。
}{
接口：\texttt{DeleteMessage}；事件：\texttt{MessageDeleted}、\texttt{UnreadUpdated}；实体：Message、AuditLog、Notification、ReadState；验收：AC-04、AC-E7。
}

\subsection{语音频道状态同步}
本版本的语音频道为状态同步 MVP，仅描述成员进入、退出和设备状态变化，不要求真实音频媒体流。

\Requirement{FR-16}{加入与退出语音频道}{P0}{
社区成员
}{
用户点击加入或离开语音频道。
}{
用户已登录；目标频道为语音频道；用户拥有查看和加入该语音频道的权限。
}{
系统创建或释放语音会话状态；同一用户在同一时刻仅保留一个有效语音频道会话；向同频道成员同步加入、退出和当前成员列表。
}{
频道不存在、频道类型错误、无权加入或连接状态异常时拒绝加入；断线超时后应释放会话并广播离开事件。
}{
接口：\texttt{JoinVoiceChannel}、\texttt{LeaveVoiceChannel}；事件：\texttt{VoiceMemberJoined}、\texttt{VoiceMemberLeft}；实体：VoiceSession、Channel、Membership、AuditLog；验收：AC-03、AC-E8。
}

\Requirement{FR-17}{语音状态同步}{P0}{
语音频道成员
}{
用户在语音频道内切换静音、闭麦、取消静音或连接状态。
}{
用户已处于某个有效语音频道会话中；目标状态属于系统允许的语音状态集合。
}{
系统更新成员语音状态；向同频道成员广播状态变化；客户端成员列表展示一致的静音、闭麦和连接状态。
}{
无有效语音会话、状态值非法或用户无权继续停留频道时拒绝更新；权限被移除后应自动退出语音状态。
}{
接口：\texttt{UpdateVoiceState}；事件：\texttt{VoiceStateChanged}、\texttt{VoiceMemberLeft}；实体：VoiceSession、PermissionOverwrite、AuditLog；验收：AC-03、AC-N2。
}

\subsection{通知与提醒}
本模块用于保障用户能够及时获知与其相关的重要交互事件。

\Requirement{FR-18}{通知生成与已读管理}{P0}{
注册用户、社区成员、好友
}{
系统发生好友申请、私聊新消息、频道提及、管理动作结果或未读变化。
}{
目标用户存在；目标用户具备接收该事件通知的条件；事件未被用户通知设置屏蔽。
}{
系统为目标用户生成通知记录；更新未读计数；通过实时事件推送通知摘要；用户查看后可将通知标记为已读。
}{
无权接收事件、来源数据已失效或重复事件被去重时不得生成重复通知；标记他人通知为已读时拒绝操作。
}{
接口：\texttt{MarkNotificationRead}；事件：\texttt{NotificationCreated}、\texttt{UnreadUpdated}；实体：Notification、ReadState、Message、Friendship；验收：AC-05、AC-E6。
}

\subsection{角色与权限控制}
本模块用于定义社区内的角色能力边界和具体权限校验行为。

\Requirement{FR-19}{角色定义与分配}{P0}{
社区管理员、社区所有者
}{
具备权限的用户创建角色、修改角色权限或为成员分配角色。
}{
用户处于目标社区内；拥有角色管理权限；目标角色和目标成员属于同一社区。
}{
系统保存角色名称、优先级、颜色和权限集合；维护成员角色映射；权限变更后影响频道访问、消息操作、语音状态和管理动作。
}{
普通成员不得配置角色；管理员不得授予超出自身能力的角色；默认角色不得被删除到社区无默认权限状态。
}{
接口：\texttt{AssignRole}；事件：\texttt{PermissionChanged}、\texttt{MemberChanged}；实体：Role、Membership、PermissionOverwrite、AuditLog；验收：AC-04。
}

\Requirement{FR-20}{权限校验与拦截}{P0}{
所有用户角色
}{
用户执行查看频道、发送消息、加入语音、删除消息、管理成员、分配角色或访问附件等受控操作。
}{
系统已定义操作所需权限；请求中包含可识别的操作者、目标资源和上下文。
}{
系统在执行业务动作前统一计算有效权限；权限满足时继续执行；权限不足时拒绝请求、返回明确提示并记录必要审计。
}{
任何接口、实时事件、附件访问或历史消息加载路径均不得绕过权限校验；权限规则冲突时按所有者特权、显式拒绝优先和角色优先级规则处理。
}{
接口：\texttt{CheckPermission}；事件：\texttt{PermissionChanged}；实体：Role、PermissionOverwrite、Membership、AuditLog；验收：AC-04、AC-N4、AC-E5、AC-E7。
}
