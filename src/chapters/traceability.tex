\subsection{软件工程方法}
本项目采用以敏捷迭代为主，原型法、需求追踪、模块化开发和验收驱动测试相结合的软件工程方法。开发过程围绕 P0、P1 和 P2 优先级组织，优先完成具备演示和交付价值的 P0 MVP 范围，再根据时间与质量情况迭代补充 P1 增强能力。

\begin{itemize}
  \item 敏捷迭代开发：按照账号、社交与私聊、社区与频道、消息与附件、通知与未读、权限和语音状态等模块划分迭代任务。每轮迭代应形成可运行、可联调、可验收的阶段成果。
  \item 原型法：在实现核心界面和流程前，先通过页面原型确认社区列表、频道导航、消息收发、权限配置和通知提醒等关键交互，降低需求理解偏差和返工风险。
  \item 需求追踪：以功能需求编号为主线，维护需求、开发模块、接口与事件、关键实体和验收编号之间的映射关系，保证 P0 需求均能追踪到实现和测试依据。
  \item 模块化开发：各模块保持清晰职责边界，公共权限校验、实时事件分发、日志审计和配置限制作为跨模块支撑能力统一处理，避免重复实现和规则不一致。
  \item 验收驱动测试：开发完成后以验收编号、异常流程、权限矩阵和非功能指标作为测试依据，重点检查注册登录、社区加入、频道消息、角色权限、通知未读和语音状态同步等核心场景。
  \item 版本控制与联调：使用 \texttt{Git} 管理代码版本，按功能或模块进行阶段性提交与合并，并在每轮迭代后开展接口联调和回归检查，降低集成风险。
\end{itemize}
